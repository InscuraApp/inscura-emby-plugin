# Inscura の Emby メタデータプラグイン

言語： [English](../README.md) | [简体中文](README-zh.md) | **日本語** | [한국어](README-ko.md)

Inscura は、映画情報、キャスト、ジャンル、コレクション、カバー画像、背景画像、アルバム写真、予告編、メディアストリームなどのデータを整理できるローカルメディアライブラリアプリです。Emby はメディアライブラリの再生と管理を担当しますが、Inscura で整理されたこれらのデータについては認識していません。

このプラグインは、InscuraのローカルAPIサービスをEmbyに連携させるためのものです。Embyが映画をスキャンまたは更新する際、プラグインはタイトル、概要、公開日、評価、ジャンル、タグ、コレクション、キャスト、監督、脚本家、プロデューサー、画像、ストリーミング情報、およびYouTubeの予告編をEmbyのメタデータに書き込みます。

このプラグインは、Inscuraのメディアライブラリデータとライブラリ内で生成された画像リソースのみを読み取り、元のメディアファイルのダウンロード、移動、名前変更、削除、または変更は行いません。

## 現在の機能

- Emby から渡された実際のファイルパスとファイル名を優先して照合し、Emby のタイトルが手動で変更されたことによる誤った照合を減らします。
- パスによる照合が見つからない場合、番号、ファイル名に含まれる識別コード、および Emby のタイトルによる照合を順次試みます。
- Embyが受け入れる映画メタデータフィールドへの書き込みに対応しています。これには、タイトル、オリジナルタイトル、ソート用タイトル、概要、キャッチコピー、公開日、年、評価、コンテンツレーティング、制作会社、ジャンル、国、タグ、コレクション、キャスト、監督、脚本家、プロデューサー、外部IDが含まれます。
- メディアストリーム情報のインポートに対応しています。これには、動画、音声、字幕ストリームのエンコード、解像度、フレームレート、ビットレート、言語、タイトル、音声トラック、デフォルト／強制字幕のマークなどの情報が含まれます。
- Embyのさまざまな映画画像タイプ（ポスター、背景画像、サムネイル、バナー、ロゴ、アートワーク、ディスクジャケットなど）のインポートに対応しています。
- メタデータの更新後に、Embyで欠落している画像タイプを自動的に補完する機能をサポートしています。デフォルトでは、欠落している画像のみを補完し、Embyにすでに存在する画像を上書きすることはありません。
- 背景画像には、優先的に Inscura の横長カバー画像／背景画像が使用され、次にプレビュー画像、動画のスクリーンショット、アルバム内のデフォルトタイプの画像が追加されます。
- 俳優、監督、脚本家、プロデューサーなどの人物関係をインポートでき、俳優についてはプロフィール画像、プロフィール、誕生日、国／地域、俳優タグをインポートできます。
- リモート予告編は現在、YouTubeのURLのみインポート可能です。YouTube以外のローカルまたはオンライン予告編については、InscuraアプリのNFO機能でエクスポートした後、Embyにスキャンさせてください。

## InscuraのローカルAPIサービスを有効にする

1. Inscuraを起動し、Embyと同期したいメディアライブラリを開きます。
2. 設定内のAPIサービス設定に移動し、ローカルAPIサービスを有効にします。
3. 認証方式はトークンモードのままにしておくことを推奨します。また、設定ページに表示されるAPIトークンを保存してください。
4. Emby サーバーが動作しているデバイスでヘルスチェック URL にアクセスし、サービスに接続できることを確認します。

例：

```bash
curl "http://[ip]:28687/api/v1/health"
```

Emby と Inscura が同じマシン上にない場合、プラグインのサービスアドレスには `127.0.0.1` を入力せず、Inscura を実行しているコンピュータの LAN 内でのアドレスを入力してください。例：

```text
http://[ip]:28687
```

ローカルインターフェースサービスは、現在のメディアライブラリのライフサイクルに合わせて動作します。メディアライブラリが開いており、サービスが有効になっている間はポートをリッスンし、**メディアライブラリがロックされたり、閉じられたり、アプリケーションが終了したりすると、サービスは停止します。**

## プラグインのダウンロード

[GitHub](https://github.com/InscuraApp/inscura-emby-plugin/archive/refs/heads/main.zip) または [Releases](https://github.com/InscuraApp/inscura-emby-plugin/releases) から最新のプラグインファイルをダウンロードしてください。

Releases に掲載されている、あらかじめビルド済みのプラグインパッケージを優先してダウンロードすることをお勧めします。プラグインのインストールファイルは以下の通りです：

| ファイル | 用途 |
| --- | --- |
| `Emby.Plugin.Inscura.dll` | Emby プラグイン本体。検索、マッチング、メタデータの書き込み、画像および予告編のインポートを担当 |

ソースコードパッケージをダウンロードした場合は、ローカルでビルドすることも可能です：

```bash
cd inscura-emby-plugin
dotnet build -c Release
```

ビルド結果は以下の場所にあります：

```text
bin/Release/net8.0/Emby.Plugin.Inscura.dll
```

## プラグインのインストール

`Emby.Plugin.Inscura.dll` を Emby Server のプラグインディレクトリにコピーし、Emby Server を再起動してください。

Emby のプラグインディレクトリは、システム、パッケージのソース、コンテナのマッピング、およびインストール方法によって異なります。以下に一般的な場所を挙げますが、お使いの環境がこれと異なる場合は、Emby Server のコンソール、パッケージページ、コンテナのマッピング、またはシステム内の実際のデータディレクトリを参照してください。

| 環境 | 一般的なプラグインディレクトリ |
| --- | --- |
| Windows | `%APPDATA%\Emby-Server\programdata\plugins` |
| Linux | `/var/lib/emby/plugins` |
| macOS | `~/.config/emby-server/plugins` |
| Docker | コンテナ内の `/config/plugins`、またはホストマシンからマッピングされた `config/plugins` |
| Synology パッケージ | 実際のパッケージデータディレクトリ（例：`/vol1/@appdata/EmbyServer4-9/plugins`）を基準とします |

インストール手順：

1. Emby Server を停止するか、再起動の準備をします。
2. `Emby.Plugin.Inscura.dll` を Emby プラグインディレクトリに配置します。
3. Emby の実行ユーザーが当該ファイルを読み取る権限を持っていることを確認します。
4. Emby Server を再起動します。
5. Embyの管理画面を開き、プラグイン一覧またはプラグイン設定ページで `Inscura` が表示されていることを確認します。

## Embyでプラグインを有効にする

1. Embyの管理画面にアクセスします。
2. プラグイン設定ページを開き、`Inscura` を選択します。
3. InscuraのAPIアドレスとAPIトークンを入力します。
4. 必要に応じて、映画メタデータのスクレイピング、画像のインポート、画像タイプの自動補完、キャストのメタデータとアバターのインポート、YouTube予告編、メディアストリーム情報などのオプションを有効にします。
5. 映画ライブラリの設定を開き、「メタデータダウンローダー」および「画像ダウンローダー」で `Inscura` を有効にします。
6. Emby が Inscura のデータを優先して使用するように、`Inscura` をリストの上位に配置することをお勧めします。
7. 設定を保存します。
8. 映画ライブラリまたは個々の作品に対して、メタデータの更新を実行します。初回検証時は、まず少数の作品から更新することをお勧めします。

## プラグイン設定の説明

| 設定 | 説明 |
| --- | --- |
| インターフェース言語 | プラグイン設定ページの表示言語。英語と簡体字中国語に対応 |
| Inscura API アドレス | Inscura ローカルインターフェースサービスのアドレス。Emby と Inscura が同じマシンにない場合は、LAN アドレスを入力する必要があります |
| API トークン | ローカルインターフェースサービスでトークン認証を使用する場合に入力します。Inscura で認証なしに設定されている場合は、空欄のままにできます |
| 検索候補数 | 1回のマッチングごとに Inscura からリクエストする候補の数 |
| リクエストタイムアウト | プラグインが Inscura API にアクセスする際のタイムアウト時間（単位：秒） |
| 映画メタデータのスクレイピングを有効にする | 有効にすると、Embyが映画メタデータを更新する際にInscuraから映画情報を取得します |
| 画像のインポートを有効にする | 有効にすると、EmbyはInscuraから映画の画像を取得できます |
| Emby に欠落している画像タイプの自動補完 | 有効にすると、映画メタデータの更新が完了した際、プラグインが Emby に欠落しているポスター、背景画像、サムネイル、バナー画像、ロゴ、アートワーク、ディスクジャケットを自動的に保存します |
| 出演者のメタデータとアバターのインポートを有効化 | 有効にすると、EmbyはInscuraから人物情報と人物画像を取得できます |
| YouTubeの予告編をインポート | 有効にすると、プラグインはYouTube形式のリモート予告編のみをインポートします |
| Inscuraのプレビュー画像をサムネイル候補として使用 | 有効にすると、Inscuraのプレビュー画像がEmbyのサムネイル候補として使用されます |
| ギャラリー画像とスクリーンショットを背景画像／スクリーンショットの候補として扱う | 有効にすると、Inscuraの背景画像、プレビュー画像、動画のスクリーンショット、およびギャラリーのデフォルト画像が、背景画像またはスクリーンショットの候補として扱われます |
| Inscuraのメディアストリーム情報をインポート | 有効にすると、プラグインはInscuraに記録された動画、音声、字幕ストリーム情報をEmbyに提供します |

## 画像タイプの対応関係

| Inscuraの画像タイプ | Embyの画像タイプ |
| --- | --- |
| ポスター / poster | Primary |
| 横型カバー、背景画像、ギャラリーのデフォルト画像 / fanart | Backdrop |
| 動画のスクリーンショット / screenshot | Screenshot |
| 横型画像 / landscape | Thumb |
| プレビュー画像 / preview | Thumb、Backdropの候補にも含まれる |
| バナー画像 / banner | Banner |
| ロゴ / clearlogo | Logo |
| アートワーク / clearart、keyart | Art |
| ディスクジャケット / discart | Disc |

## 使用上のアドバイス

- 初めて使用する際は、まず少数の作品を選んでメタデータを更新し、タイトル、出演者、画像、コレクションが期待どおりであることを確認してから、ライブラリを一括更新することをお勧めします。
- Emby内で作品のタイトルが手動で変更されていた場合でも、プラグインはタイトルを唯一の根拠とせず、実際のファイルパスとファイル名による照合を優先します。
- Inscuraのサービスアドレスやトークンが変更された場合は、Embyプラグインの設定で同期更新を行い、その後メタデータを更新する必要があります。
- Inscuraのメディアライブラリがロックされているか停止されている場合、Embyはメタデータを読み取ることができません。
- プラグインをアップグレードした後も設定ページに古い内容が表示される場合は、まずブラウザのページを強制リロードしてから、プラグインの設定ページを再度開いてください。

## トラブルシューティング

### Emby で Inscura プラグインが表示されない

1. `Emby.Plugin.Inscura.dll` が Emby プラグインディレクトリにあることを確認してください。
2. Emby プロセスが当該ファイルを読み取る権限を持っていることを確認してください。
3. 現在の Emby Server バージョンが `.NET 8` プラグインに対応していることを確認してください。
4. Emby Server を再起動してください。
5. EmbyのWebインターフェースを再起動し、プラグイン一覧または設定ページで確認してください。
6. それでもプラグインが表示されない場合は、Emby Serverのログでプラグインの読み込みエラーを確認してください。

### Embyでプラグインは表示されるが、メタデータが表示されない

1. Embyサーバーが動作しているデバイスで、InscuraのヘルスチェックURLにアクセスしてください。
2. Inscura メディアライブラリが起動しており、ローカル API サービスが有効になっていることを確認してください。
3. Emby プラグイン内の API アドレスが誤った `127.0.0.1` になっていないことを確認してください。
4. API がトークン認証を使用している場合は、Emby プラグインに正しいトークンが入力されていることを確認してください。
5. 映画メディアライブラリのメタデータダウンローダーで `Inscura` が有効になっていることを確認してください。
6. 映画に対して、メタデータの更新または再認識を実行してください。

### 画像やキャストのアバターが表示されない

1. Emby サーバーが Inscura API アドレスにアクセスできることを確認してください。
2. インターフェースがトークン認証を使用している場合は、プラグインのトークンが正しいことを確認してください。
3. プラグインの設定で、画像のインポート、キャストのメタデータ、およびアバターのインポートが有効になっていることを確認してください。
4. Inscura 内で、該当するメディアまたはキャストに利用可能な画像リソースが実際に存在することを確認してください。
5. 該当する作品または人物に対して、メタデータの更新を実行してください。

### バナー画像、ディスクジャケット、アートワーク、または複数の背景画像がインポートされない

1. プラグインの設定で「画像のインポート」が有効になっていることを確認してください。
2. プラグインの設定で、Embyに欠落している画像タイプの自動補完が有効になっていることを確認してください。
3. Inscura内の対応するアルバム画像が正しいタイプに設定されていることを確認してください。
4. 作品に対してメタデータの更新を実行し、更新完了後にプラグインが欠落している画像を保存するようにします。
5. Emby内に同タイプの画像がすでに存在する場合、プラグインはデフォルトでは既存の画像を上書きしません。

### 予告編がインポートされていない

1. プラグインの設定で、YouTube 予告編のインポートが有効になっていることを確認してください。
2. Inscura 内の予告編のアドレスが YouTube の URL であることを確認してください。
3. YouTube 以外のローカルまたはオンラインの予告編は、このプラグインでは直接インポートされません。Inscura アプリの NFO 機能を使用してエクスポートし、Emby にスキャンさせてください。

## プラグインのアップグレード

1. Emby Serverを停止するか、再起動の準備をしてください。
2. Embyのプラグインディレクトリ内の古いファイルを、新しい `Emby.Plugin.Inscura.dll` で上書きしてください。
3. Embyを実行しているユーザーが新しいファイルを読み取る権限を持っていることを確認してください。
4. Emby Serverを再起動します。
5. Embyのウェブ版を再度開き、プラグイン一覧またはプラグイン設定ページでバージョンと設定を確認します。

アップグレード後もEmbyのウェブ版に古い設定が表示される場合は、まずブラウザのページを強制リロードしてから、プラグイン設定ページを再度開いてください。
