# Inscura의 Emby 메타데이터 플러그인

언어: [English](../README.md) | [简体中文](README-zh.md) | [日本語](README-ja.md) | **한국어**

Inscura는 영화 정보, 출연진, 장르, 컬렉션, 커버, 배경 이미지, 앨범 사진, 예고편 및 미디어 스트림 등의 자료를 정리할 수 있는 로컬 미디어 라이브러리 애플리케이션입니다. Emby는 미디어 라이브러리의 재생 및 관리를 담당하지만, Inscura에서 이미 정리해 둔 이러한 데이터에 대해서는 알지 못합니다.

이 플러그인은 Inscura의 로컬 API 서비스를 Emby에 연동하는 데 사용됩니다. Emby가 영화를 스캔하거나 새로 고칠 때, 플러그인은 제목, 소개, 개봉일, 평점, 장르, 태그, 컬렉션, 출연진, 감독, 각본가, 제작자, 이미지, 미디어 스트림 정보 및 YouTube 예고편을 Emby 메타데이터에 기록합니다.

이 플러그인은 Inscura의 미디어 라이브러리 데이터와 라이브러리 내에서 생성된 이미지 리소스만 읽을 뿐, 원본 미디어 파일을 다운로드, 이동, 이름 변경, 삭제 또는 수정하지 않습니다.

## 현재 기능

- Emby에서 전달된 실제 파일 경로와 파일명을 우선적으로 매칭하여, Emby 제목이 수동으로 수정되어 발생하는 오매칭을 줄입니다.
- 경로 매칭이 불가능할 경우, 번호, 파일명 내 식별 코드 및 Emby 제목을 순차적으로 시도합니다.
- 제목, 원제, 정렬 제목, 소개, 슬로건, 개봉일, 연도, 평점, 콘텐츠 등급, 제작사, 장르, 국가, 태그, 컬렉션, 배우, 감독, 각본가, 프로듀서 및 외부 ID를 포함하여 Emby가 수용 가능한 영화 메타데이터 필드에 데이터를 기록할 수 있습니다.
- 비디오, 오디오 및 자막 스트림의 인코딩, 해상도, 프레임 속도, 비트레이트, 언어, 제목, 오디오 채널, 기본/강제 자막 태그 등의 미디어 스트림 정보 가져오기를 지원합니다.
- Emby의 다양한 영화 이미지 유형(포스터, 배경 이미지, 썸네일, 배너, 로고, 아트웍, 디스크 커버 등)의 가져오기를 지원합니다.
- 메타데이터가 갱신된 후 Emby에 누락된 이미지 유형을 자동으로 보완합니다. 기본적으로 누락된 이미지만 보완하며, Emby에 이미 존재하는 이미지는 덮어쓰지 않습니다.
- 배경 이미지는 우선 Inscura의 가로형 커버/배경 이미지를 사용하며, 그 다음으로 미리보기 이미지, 동영상 캡처 이미지 및 앨범에 있는 기본 유형의 이미지를 추가합니다.
- 배우, 감독, 각본가, 제작자 등의 인물 관계를 가져올 수 있으며, 배우의 경우 프로필 사진, 소개, 생일, 국가/지역 및 배우 태그를 가져올 수 있습니다.
- 원격 예고편은 현재 YouTube 주소만 가져옵니다. YouTube가 아닌 로컬 또는 온라인 예고편은 Inscura 앱의 NFO 기능을 통해 내보낸 후 Emby에서 스캔하도록 하십시오.

## Inscura 로컬 API 서비스 활성화

1. Inscura를 실행하고 Emby와 동기화할 미디어 라이브러리를 엽니다.
2. 설정 메뉴의 API 서비스 설정으로 이동하여 로컬 API 서비스를 활성화합니다.
3. 인증 방식은 토큰 모드로 유지하는 것이 좋으며, 설정 페이지에 표시된 API 토큰을 저장해 두십시오.
4. Emby 서버가 설치된 기기에서 상태 확인 주소에 접속하여 서비스에 접속 가능한지 확인합니다.

예시:

```bash
curl "http://[ip]:28687/api/v1/health"
```

Emby와 Inscura가 같은 컴퓨터에 설치되어 있지 않은 경우, 플러그인의 서비스 주소란에 `127.0.0.1`을 입력해서는 안 되며, Inscura가 실행 중인 컴퓨터의 LAN 내 주소를 입력해야 합니다. 예:

```text
http://[ip]:28687
```

로컬 인터페이스 서비스는 현재 미디어 라이브러리의 수명 주기에 따라 실행됩니다. 미디어 라이브러리가 열려 있고 서비스가 활성화된 상태에서는 포트를 감시하며, **미디어 라이브러리가 잠기거나 닫히거나 애플리케이션이 종료되면 서비스가 중지됩니다.**

## 플러그인 다운로드

[GitHub](https://github.com/InscuraApp/inscura-emby-plugin/archive/refs/heads/main.zip) 또는 [Releases](https://github.com/InscuraApp/inscura-emby-plugin/releases)에서 최신 플러그인 파일을 다운로드하십시오.

Releases에 있는 미리 빌드된 플러그인 패키지를 우선적으로 다운로드하는 것을 권장합니다. 플러그인 설치 파일은 다음과 같습니다:

| 파일 | 용도 |
| --- | --- |
| `Emby.Plugin.Inscura.dll` | Emby 플러그인의 핵심 구성 요소로, 검색, 매칭, 메타데이터 작성, 이미지 및 예고편 가져오기를 담당합니다 |

소스 코드 패키지를 다운로드한 경우, 로컬에서 빌드할 수도 있습니다:

```bash
cd inscura-emby-plugin
dotnet build -c Release
```

빌드 결과물은 다음 위치에 있습니다:

```text
bin/Release/net8.0/Emby.Plugin.Inscura.dll
```

## 플러그인 설치

`Emby.Plugin.Inscura.dll`을 Emby Server의 플러그인 디렉터리에 복사한 후, Emby Server를 재시작합니다.

Emby 플러그인 디렉터리는 시스템, 패키지 출처, 컨테이너 매핑 및 설치 방식에 따라 달라집니다. 아래에는 일반적인 위치가 나열되어 있습니다. 사용자의 환경이 이와 다를 경우, Emby Server 콘솔, 패키지 페이지, 컨테이너 매핑 또는 시스템의 실제 데이터 디렉터리를 기준으로 삼으십시오.

| 환경 | 일반적인 플러그인 디렉터리 |
| --- | --- |
| Windows | `%APPDATA%\Emby-Server\programdata\plugins` |
| Linux | `/var/lib/emby/plugins` |
| macOS | `~/.config/emby-server/plugins` |
| Docker | 컨테이너 내 `/config/plugins` 또는 호스트 머신에 매핑된 `config/plugins` |
| Synology 패키지 | 실제 패키지 데이터 디렉터리를 기준으로 하며, 예: `/vol1/@appdata/EmbyServer4-9/plugins` |

설치 단계:

1. Emby Server를 중지하거나 재시작할 준비를 합니다.
2. `Emby.Plugin.Inscura.dll` 파일을 Emby 플러그인 디렉터리에 넣습니다.
3. Emby 실행 사용자가 해당 파일에 대한 읽기 권한이 있는지 확인합니다.
4. Emby Server를 재시작합니다.
5. Emby 관리자 콘솔을 열고, 플러그인 목록 또는 플러그인 설정 페이지에서 `Inscura`가 표시되는지 확인합니다.

## Emby에서 플러그인 활성화

1. Emby 관리자 콘솔에 접속합니다.
2. 플러그인 설정 페이지를 열고 `Inscura`로 이동합니다.
3. Inscura API 주소와 API 토큰을 입력합니다.
4. 필요에 따라 영화 메타데이터 스크래핑, 이미지 가져오기, 이미지 유형 자동 완성, 배우 메타데이터 및 프로필 사진 가져오기, YouTube 예고편, 미디어 스트림 정보 등의 옵션을 활성화합니다.
5. 영화 미디어 라이브러리 설정을 열고, 메타데이터 다운로더 및 이미지 다운로더에서 `Inscura`를 활성화합니다.
6. Emby가 Inscura의 데이터를 우선적으로 사용하도록 `Inscura`를 순서 상단에 배치하는 것이 좋습니다.
7. 설정을 저장합니다.
8. 영화 라이브러리 또는 개별 영화에 대해 메타데이터 새로 고침을 실행합니다. 첫 검증 시에는 소수의 영화만 먼저 새로 고치는 것이 좋습니다.

## 플러그인 설정 설명

| 설정 | 설명 |
| --- | --- |
| 인터페이스 언어 | 플러그인 구성 페이지의 표시 언어. 영어와 중국어(간체) 지원 |
| Inscura API 주소 | Inscura 로컬 인터페이스 서비스 주소. Emby와 Inscura가 같은 서버에 있지 않을 경우 반드시 LAN 주소를 입력해야 합니다 |
| API 토큰 | 로컬 인터페이스 서비스에서 토큰 인증을 사용할 때 입력합니다. Inscura가 인증 없음으로 설정된 경우 비워둘 수 있습니다 |
| 검색 후보 수 | 매번 일치 검색 시 Inscura에서 요청하는 후보 수 |
| 요청 타임아웃 | 플러그인이 Inscura API에 접속할 때의 타임아웃 시간(단위: 초) |
| 영화 메타데이터 스크래핑 활성화 | 활성화하면 Emby가 영화 메타데이터를 새로 고칠 때 Inscura에서 영화 정보를 가져옵니다 |
| 이미지 가져오기 활성화 | 활성화하면 Emby가 Inscura에서 영화 이미지를 가져올 수 있습니다 |
| Emby에 누락된 이미지 유형 자동 보완 | 활성화하면 영화 메타데이터 새로 고침이 완료될 때 플러그인이 Emby에 누락된 포스터, 배경 이미지, 썸네일, 배너, 로고, 아트웍 및 디스크 커버를 자동으로 저장합니다 |
| 배우 메타데이터 및 프로필 사진 가져오기 활성화 | 활성화하면 Emby가 Inscura에서 배우 정보와 사진을 가져올 수 있습니다 |
| YouTube 예고편 가져오기 | 활성화하면 플러그인은 YouTube 형식의 원격 예고편만 가져옵니다 |
| Inscura 미리보기 이미지를 썸네일 후보로 지정 | 활성화하면 Inscura 미리보기 이미지가 Emby 썸네일 후보로 지정됩니다 |
| 갤러리 이미지와 스크린샷을 배경/스크린샷 후보로 사용 | 활성화하면 Inscura의 배경 이미지, 미리보기 이미지, 동영상 스크린샷 및 갤러리의 기본 이미지가 배경 이미지 또는 스크린샷 후보로 사용됩니다 |
| Inscura 미디어 스트림 정보 가져오기 | 활성화하면 플러그인이 Inscura에 기록된 동영상, 오디오 및 자막 스트림 정보를 Emby에 제공합니다 |

## 이미지 유형 대응 관계

| Inscura 이미지 유형 | Emby 이미지 유형 |
| --- | --- |
| 포스터 / poster | Primary |
| 가로형 커버, 배경 이미지, 갤러리 기본 이미지 / fanart | Backdrop |
| 동영상 스크린샷 / screenshot | Screenshot |
| 가로형 이미지 / landscape | Thumb |
| 미리보기 이미지 / preview | Thumb, Backdrop 후보로도 포함됨 |
| 배너 / banner | Banner |
| 로고 / clearlogo | Logo |
| 아트웍 / clearart, keyart | Art |
| 디스크 커버 / discart | Disc |

## 사용 권장 사항

- 처음 사용할 때는 소량의 영상을 선택하여 메타데이터를 갱신한 후, 제목, 출연진, 이미지 및 컬렉션이 예상대로 표시되는지 확인한 다음, 데이터베이스를 일괄 갱신하는 것이 좋습니다.
- Emby에서 영상 제목이 수동으로 변경된 적이 있더라도, 플러그인은 제목을 유일한 기준으로 삼지 않고 실제 파일 경로와 파일 이름 일치를 우선적으로 사용합니다.
- Inscura 서비스 주소나 토큰이 변경된 경우, Emby 플러그인 설정에서 이를 동기화하여 업데이트한 후 메타데이터를 새로 고쳐야 합니다.
- Inscura 미디어 라이브러리가 잠겨 있거나 닫혀 있는 경우, Emby는 메타데이터를 읽을 수 없습니다.
- 플러그인을 업그레이드한 후에도 설정 페이지에 이전 내용이 표시된다면, 먼저 브라우저 페이지를 강제 새로고침한 다음 플러그인 설정 페이지를 다시 열어주세요.

## 문제 해결

### Emby에서 Inscura 플러그인이 보이지 않는 경우

1. `Emby.Plugin.Inscura.dll` 파일이 Emby 플러그인 디렉터리에 있는지 확인하십시오.
2. Emby 프로세스가 해당 파일을 읽을 수 있는 권한이 있는지 확인하십시오.
3. 현재 Emby Server 버전이 `.NET 8` 플러그인을 지원하는지 확인하십시오.
4. Emby Server를 재시작하십시오.
5. Emby 웹 버전을 다시 열고, 플러그인 목록 또는 설정 페이지에서 확인하십시오.
6. 여전히 플러그인이 보이지 않으면, Emby Server 로그에서 플러그인 로딩 오류를 확인하십시오.

### Emby에서 플러그인은 표시되지만 메타데이터가 없는 경우

1. Emby 서버가 설치된 기기에서 Inscura 상태 확인 주소에 접속하십시오.
2. Inscura 미디어 라이브러리가 열려 있고, 로컬 API 서비스가 실행 중인지 확인하십시오.
3. Emby 플러그인의 API 주소가 잘못된 `127.0.0.1`이 아닌지 확인하십시오.
4. API가 토큰 인증을 사용하는 경우, Emby 플러그인에 올바른 토큰이 입력되었는지 확인하십시오.
5. 영화 미디어 라이브러리의 메타데이터 다운로더에서 `Inscura`가 활성화되어 있는지 확인하십시오.
6. 해당 영화에 대해 메타데이터 새로 고침 또는 재인식을 실행하십시오.

### 이미지 또는 배우 프로필 사진이 표시되지 않음

1. Emby 서버가 Inscura API 주소에 접근할 수 있는지 확인하십시오.
2. 인터페이스가 토큰 인증을 사용하는 경우, 플러그인 토큰이 올바른지 확인하십시오.
3. 플러그인 설정에서 이미지 가져오기, 배우 메타데이터 및 프로필 사진 가져오기가 활성화되어 있는지 확인하십시오.
4. Inscura에서 해당 미디어나 배우에 대해 사용 가능한 이미지 리소스가 실제로 있는지 확인하십시오.
5. 해당 영화나 인물에 대해 메타데이터 새로 고침을 실행하십시오.

### 배너 이미지, 디스크 커버, 아트웍 또는 여러 장의 배경 이미지가 가져오지 않음

1. 플러그인 설정에서 이미지 가져오기가 활성화되어 있는지 확인하십시오.
2. 플러그인 설정에서 Emby에 누락된 이미지 유형을 자동으로 보완하는 기능이 활성화되어 있는지 확인하십시오.
3. Inscura에서 해당 앨범 이미지가 올바른 유형으로 설정되어 있는지 확인하십시오.
4. 영화에 대해 메타데이터 새로 고침을 실행하여, 새로 고침이 완료된 후 플러그인이 누락된 이미지를 저장하도록 하십시오.
5. Emby에 이미 동일한 유형의 단일 이미지가 있는 경우, 플러그인은 기본적으로 기존 이미지를 덮어쓰지 않습니다.

### 예고편이 가져오지 않음

1. 플러그인 설정에서 YouTube 예고편 가져오기가 활성화되어 있는지 확인하십시오.
2. Inscura에 있는 예고편 주소가 YouTube URL인지 확인하십시오.
3. YouTube가 아닌 로컬 또는 온라인 예고편은 이 플러그인을 통해 직접 가져올 수 없으므로, Inscura 앱의 NFO 기능을 통해 내보낸 후 Emby에서 스캔하도록 하십시오.

## 플러그인 업그레이드

1. Emby 서버를 중지하거나 재시작할 준비를 하세요.
2. Emby 플러그인 디렉토리 내의 기존 파일을 새로운 `Emby.Plugin.Inscura.dll` 파일로 덮어쓰세요.
3. Emby 실행 사용자가 새 파일을 읽을 수 있는 권한이 있는지 확인하세요.
4. Emby 서버를 재시작하십시오.
5. Emby 웹 버전을 다시 열고, 플러그인 목록 또는 플러그인 설정 페이지로 이동하여 버전과 설정을 확인하십시오.

업그레이드 후에도 Emby 웹 버전에 이전 설정이 계속 표시된다면, 먼저 브라우저 페이지를 강제 새로고침한 후 플러그인 설정 페이지를 다시 열어보십시오.
